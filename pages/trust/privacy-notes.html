日本語

現在の公開プレビュー

現在のページはKazuma自身のNightscoutデータを表示する公開サンプルです。公開URLを知る人は表示内容を閲覧できます。

ユーザー版の接続情報

User Foundation 0.3では、GlurooまたはNightscoutの接続先URLと接続用の合言葉を、利用する人が選んだスマートフォンやパソコンの中だけに保存します。GlucoScopeのWorker、Kazuma個人のAzure、Cloudflare KV、Durable Objectsには保存しません。

端末内の保存は、特別な暗号化金庫ではありません。家族などと共用している端末では「この端末に保存する」をオフにするか、利用後に保存した接続を削除してください。

Apple、Google、Gluroo、Dexcom、Libreなどのログインパスワードを、GlucoScopeへ入力してもらうことはありません。

ユーザー版のAI

現在の共有AIキャッシュは公開デモ向けのため、利用者ごとの分離が完成するまで、ユーザー版からGlucoScopeのAI Workerへ血糖サマリーを送りません。ブラウザ内のいつものお話と、利用者自身がChatGPTへ渡すためのコピー機能は利用できます。

APIキーをブラウザへ置かない

OpenAI APIキーとTurnstileの秘密鍵はCloudflare WorkerのSecretとして管理し、GitHub PagesのHTMLやJavaScriptには置きません。

AIへ送る内容

AIお手紙では、表示中の期間、TIR、TAR、TBR、平均、CV、GMI、GlucoScore、傾向の要約など、振り返りに必要なサマリーをWorker経由で送ります。個人名や自由記述を必要以上に含めない設計を優先します。

利用カウンター

AIの生成回数、トークン数、推定利用額、Turnstile成功・失敗などの運用情報を保存します。血糖値やAIお手紙本文そのものを利用カウンターには保存しません。

AIお手紙のキャッシュ

保存済みAIお手紙は、閲覧したブラウザのlocalStorageと、Cloudflare Workers KVの共有キャッシュを使います。共有KVには生成済みのお手紙本文と必要最小限のメタ情報だけを保存し、送信された血糖サマリーそのものは保存しません。

同じ条件のお手紙は1時間以内なら再利用し、新しいAI生成を行いません。古いお手紙は生成失敗時などのフォールバックとして最大24時間だけ残し、その後は自動削除します。

公開ページのアクセス分析

公開デモなどの公開ページにはCloudflare Web Analyticsを導入し、ページ閲覧や表示性能の全体傾向を確認しています。ただし、ユーザーモード中だけでなく、接続情報がこのブラウザに保存されている間は、AboutやTrustを含むGlucoScope内のページでWeb Analyticsのビーコンを読み込みません。保存状態を安全に確認できない場合も読み込みません。個々の人をGlucoScope内で識別するための独自IDや、独自の行動追跡イベントも追加していません。

ユーザー版で使うChart.jsはGlucoScope内に同梱し、接続情報や血糖データを扱うページで外部CDNのグラフ用JavaScriptを実行しない構成にしています。

血糖値、GlucoScore、AIお手紙本文、Nightscout URL、API情報をアクセス分析のイベント名や追加データとして送る実装は行いません。スマホの各タブ操作も、健康情報を含む個別イベントとして追跡しません。

EN

English summary

Public preview

The current public sample displays Kazuma’s own Nightscout data to anyone who has the URL.

User connection details

In User Foundation 0.3, the Gluroo or Nightscout connection URL and passphrase stay only on the selected phone or computer. They are not stored in the GlucoScope Worker, Kazuma's Azure account, Cloudflare KV, or Durable Objects.

Device storage is not a dedicated encrypted vault. On a shared device, turn off Save on this device or remove the saved connection after use.

GlucoScope does not ask for Apple, Google, Gluroo, Dexcom, Libre, or other manufacturer login passwords.

User-mode AI

The current shared AI cache is for the public demo. Until per-user isolation is complete, user-mode summaries are not sent to the GlucoScope AI Worker. The local Gluco reflection and copy-to-ChatGPT path remain available.

Secrets stay server-side

OpenAI and Turnstile secret keys are stored as Worker secrets, not in browser code.

AI summaries

Only the summary needed for reflection is sent through the Worker. Personal identifiers should be minimized.

Operational counters

Usage counts and estimated developer cost may be stored, but glucose values and letter text are not stored in the usage counter.

AI letter cache

Saved AI letters use browser localStorage and a shared Cloudflare Workers KV cache. KV stores only the generated letter and minimal metadata, not the submitted glucose summary.

The same letter is reused for one hour without a new AI generation. A stale fallback may remain for up to 24 hours and is then deleted automatically.

Public web analytics

Public-demo pages use Cloudflare Web Analytics to review aggregate page visits and performance. The analytics beacon is not loaded in user mode, while a GlucoScope user connection remains in local or session browser storage, or when that storage state cannot be checked. This applies across same-origin About and Trust pages. GlucoScope does not add a custom visitor ID or custom behavioral events.

Chart.js is served from a reviewed local copy on the user-data page instead of running the chart library from a third-party CDN.

Glucose values, GlucoScore, AI letter text, Nightscout URLs, and API information are not intentionally sent as analytics event names or additional analytics data. Mobile tab actions are not tracked as health-data events.
